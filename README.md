# Задача:

# Пакетная обработка данных при помощи spring batch

Создать 2+ сервиса реализованных на spring-boot, которые посылают и принимают сообщение через kafka:

1. сервисы-генераторы (1+) по таймеру создают в базе данных объект событие, а затем отправляют сообщение о том, что событие создано, кто (какой сервис) является создателем;
2. 1 сервис-регистратор получает через кафку сообщения о событиях, создает в БД запись о событии, а затем отсылает ответное сообщение-подтверждение;
3. Сервисы-генераторы получают подтверждения, находят в своей БД событие и отмечают его как обработанное.

В сервисы-генераторы событий нужно добавить контроллер, который будет отображать данные о кол-во сгенерированных сообщений, сколько из них обработано, а сколько нет.

В сервис регистрации событий добавить контроллер, который будет возвращать список событий (когда, от кого, что за событие, какая-то под инфа) с поддержкой пэйджинга, фильтрацией по датам (дата начала и дата конца периода), по типу события, по источнику.

Все сервисы должны писать в лог информацию о своих действиях.

Использовать spring-batch на регистраторе для обработки очереди сообщений

# Инструкция

  Для запуска сервисов неоходимо выполнить команды
```
docker compose build
docker compose up
```

    По http://localhost:8082/api/v1/events можно посмотреть список событий с поддержкой пэйджинга, фильтрацией по датам, по типу события, по источнику.
    По http://localhost:8081/stats можно посмотреть данные о количестве сгенерированных сообщений, сколько из них обработано.
